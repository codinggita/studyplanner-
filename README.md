# 📚 Study Planner App – Full Stack Project

Welcome to the **Study Planner App!**  
This application helps students **organize their study schedule, manage tasks, and track progress efficiently** using a modern full stack architecture.

---

## 📌 Project Objective

The goal of this project is to:

- Help students **plan and manage their studies effectively**
- Provide a **structured task and schedule system**
- Improve **productivity and time management**

This application solves the problem of **unorganized study routines** by offering a centralized platform for planning and tracking.

---

## 🛠 Tech Stack Used

### Frontend
- ReactJS
- Tailwind CSS

### Backend
- Node.js
- Express.js

### Database
- MongoDB

---

## 📋 Application Features

### 🎯 Core Features

- Create and manage **study tasks**
- Set **deadlines and priorities**
- Track **daily and weekly progress**
- Mark tasks as **completed/incomplete**

---

## ⚙️ Implemented Features (As per Hackathon Requirements)

---

### 1️⃣ Routing & Navigation
- Implemented using **React Router**
- Pages included:
  - Home
  - Login
  - Signup
  - Dashboard
  - Study Planner
  - Profile / Settings

---

### 2️⃣ React Hooks Usage

- `useState` – Managing tasks and UI state  
- `useEffect` – Fetching tasks from backend  
- `useRef` – Handling input focus  
- `useContext` – Managing global user state  

---

### 3️⃣ State Management

- Implemented using **Context API**
- Used for:
  - User authentication
  - Task data sharing
  - Theme management

---

### 4️⃣ Authentication System

- User Signup & Login
- Password validation
- Authentication stored using **LocalStorage**
- Protected routes for dashboard access

---

### 5️⃣ Theme Support

- Dark Mode / Light Mode toggle
- User preference stored for persistence

---

### 6️⃣ Search, Filtering & Sorting

- Search tasks by title
- Filter by:
  - Completion status
  - Priority (High / Medium / Low)
- Sort by:
  - Deadline
  - Priority

---

### 7️⃣ Debouncing

- Implemented in:
  - Task search bar
- Optimizes performance by reducing unnecessary API calls

---

### 8️⃣ Pagination

- Implemented for task lists
- Supports:
  - Backend pagination using MongoDB (`limit`, `skip`)
  - Frontend pagination UI

---

### 9️⃣ CRUD Operations

- Create new study tasks  
- Read/display all tasks  
- Update task details  
- Delete tasks  

All operations interact with **Node.js + Express APIs** and **MongoDB database**.

---

### 🔟 API Integration

- RESTful APIs built using Express.js
- Proper error handling
- Loading indicators during API calls

---

### 1️⃣1️⃣ Form Handling & Validation

- Input validation for:
  - Task creation
  - Authentication forms
- Error messages and user feedback
- Controlled components in React

---

### 1️⃣2️⃣ Responsive UI

- Fully responsive design using Tailwind CSS
- Works across:
  - Desktop
  - Tablet
  - Mobile devices

---

### 1️⃣3️⃣ Error Handling

- Backend error responses handled properly
- Frontend displays meaningful error messages
- Try–catch used in API calls
